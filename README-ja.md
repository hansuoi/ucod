# UCOD

## 概要
- UCOD(ユーコッド, UI Class Operation Diagram)は、UMLベースのモデリング記法である。AIを活用したテスト・ドキュメンテーション・システム分析等を目的として設計されている
- 各クラスブロックは1つの「ページ」「モーダル」「コンポーネント」などを表し、以下の三層構造で記述する
    - クラス名: 画面・モーダル・UIコンポーネントの名称
    - UI要素: そのクラス内に含まれるインタラクティブまたは視覚的な要素(e.g. ボタン・テキストボックスなど)
    - ユーザーのアクション: ユーザーがその画面上で行う操作や意図を、水平線(`---`)の下に記述する
![UCODサンプル](./samples/ucod-sample.png)

## UCODの描き方
- UI要素は、`(Button)`・`(TextBox)`・`(Link)`などのプレフィックスを付けて列挙する
- あるUI要素の子要素は、インデントを用いるか、別クラスに切り出すかして表現する
    - 別クラスに切り出す場合は、`package`キーワードによるグルーピングを推奨する
- ページによって含まれているUI要素が一部異なる場合は、行頭に`+`や`-`を付けて追加・削除の差分を表現する
- クラス間の遷移は矢印(`-->`または`<->`)で表し、トリガとなるUI要素をラベルとして付ける
    - トリガが自明な場合(例: 「設定」ボタンを押すと設定ページへ遷移する等)は、ラベルを省略してもよい
- `Page`・`Modal`・`Component`などのクラス種別や、`(Button)`・`(TextBox)`などのUI要素タイプは、組織やプロジェクトの標準に合わせて自由に追加・統合してよい
- `skinparam class`による色設定も、可読性やチームの好みに応じて自由にカスタマイズしてよい
